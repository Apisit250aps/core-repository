# v0.2.0

Release date: 2026-04-20

## Highlights

- Simplified the public API to focus on `Controller` and `Repository` as the primary extension points.
- Removed dedicated `base-controller` and `base-repository` exports from both the root package and `@aps/next-api-core`.
- Replaced direct route registration `console.log` calls with a scoped logger inside the core package.
- Cleaned up TypeScript configuration by removing deprecated `baseUrl` usage and trimming internal path aliases.

## Import

```ts
import { Controller, Repository } from '@aps/next-api'
import { Controller, Repository } from '@aps/next-api-core'
```

## Breaking Changes

- Removed `@aps/next-api/base-controller` and `@aps/next-api/base-repository` subpath exports.
- Removed `@aps/next-api-core/base-controller` and `@aps/next-api-core/base-repository` subpath exports.
- Removed `BaseController` and `BaseRepository` named exports from the package entrypoint.

## Migration Notes

- Replace any direct `BaseController` usage with `Controller`.
- Replace any direct `BaseRepository` usage with `Repository`.
- Keep using `@aps/next-api-core/fields` as the canonical fields entrypoint in workspace path mappings.
